V0.3.0 | MARCH 8, 2024 | PHP, SQL, AND ARCHIVE ADDITION

General Update

It's been a while since I last updated this website. It's been a huge transition from high school to college and I often find myself busy with other projects or enjoying the city but I felt it was time to return to this vast project I adored so much. With newfound motivation, I spent hours this week learning more on PHP to finally get some knowledge on backend functionality for the website and to put my newly acquired SQL knowledge to good use. I hope to keep this motivation going for further updates. I'm excited to learn more and use my new skills in hopes of seeing this website on a live domain soon.

  • Website is now connected to a local PHP server and a SQL database
    • Database contains a list of all projects for better organization and ease of access. Currently there are three tables.
      • The projects table contains information on each project added to the website including name, category, access type, version, date created/updated, etc.
      • The categories and access_types tables list all types of each respective table that are linked with the projects table through a foreign key. These will be used for the filter feature on the products page to keep track of all types of projects without having to manually add them to the filter dropdown.
      • More tables may be added to the future if needed. Some possible plans may be adding a table for each version on this page.
      • I plan to added more columns to the projects table for more in-depth details such as descriptions, links, or images.
    • PHP will allow for easy access to this database and backend functionality for the website
      • Home Page index.html has been converted to index.php
      • Featured sections on the Home Page will now contain projects directly from the database. SQL will select a featured project of my choice (from Project_ID), the newest project, and the newly updated project. The last two are automatically updated when projects are created/updated in the database through an SQL query.
      • As of now, this is all PHP manages. I have further plans to continue using PHP for other purposes, but my main goal was to get my local server running and access the database.
  • Introducing a website archive feature here on the Version History page
    • From now on, for each update will include a snapshot of the website that can be previewed
    • Note that you will not be able to open an archive while already viewing an archive to save on space and avoid confusion
    • Also note that I will not be keeping archives of the database, so outdated versions that use the database may still display up-to-date project details
    • The idea of these archive previews will be to see the website as it was at the given version to see how the website has changed over time
    • Links to archive previews will be added to the bottom of each version description from now on
    • Also added a couple other older versions of the website to the bottom of this version update
    • The feature works great but some concerns I may have in the future is the file size of the website considering I'm essentially carrying multiple copies of the website. Right now, the website is less than 500 KB so it shouldn't be a problem for a while but it's something to keep in mind for the future.

V0.2.2 | DECEMBER 14, 2021 | JAVASCRIPT OPTIMIZATION

General Update

  • Further optimized coding format of the website using JavaScript
    • Seperated main script into sections using comments for easy developer access
    • Added the Constant Website Variables section in the main script
      • rootDirec - Defines the current root directory of the website files. When the website files are moved to a different directory, simply change this variable to update all universal links and images that use the root directory.
      • version - Defines the current version of the website. Simply change this varible to update the current version to the correct number to display in elements such as the footer.
      • siteName - Defines the current site name (Project Central as of now). Changing this variable will update all instances (except the logo). This is in case I decide to change the name later.
    • Multiple elements such as the navigation bar, footer, and favicon are now coded in JavaScript to be more adaptive
      • Now any updates to the nav bar, footer, or favicon can be displayed to all linked pages.
        • The favicon image and links in the nav bar and footer are now pathed from the root of the folder. As of now, this means it starts from the flash drive at /media/removable/USB Drive/ on Chromebook. If the directory of the website changes, simply change the rootDIrec varible and the website will be functional again.
        • The nav bar is now adaptive to each page. If the page you are currently viewing is on the nav bar, it will be highlighted.
        • The browser tab title is now adaptive to the website name. The siteName variable will be added to the end of an browser tab title (the <title> tag in HTML).
    • Version history is now also adaptive and the most recent version will open automatically on page load instead of having to manually assign the recent ID.

V0.2.0 | NOVEMBER 16, 2021 | VERSION HISTORY

Version History Page | versionhistory.html

  • Completed Version History page
    • Added a universal collapsible bar; can be adapted for any content along with collapsible items (no bar).
      • Collapsible bars marked with the "recent" ID will load already opened (for most recent version).
    • Version number, as of now, is in VX.Y.Z format.
      • X refers to the version of the overall website. Will remain at 0 until the website is completed and usable.
      • Y refers to the number of significant updates to the website. This is mostly used for when new pages are added.
      • Z refers to any other small updates added to the website such as debugging or slight modifications.
      • Note: If a version number changes, all less important numbers reset back to 0. For example, if the version is V1.2.3 and a new page is added, the new version would be V1.3.0.
    • New versions will be added with title format: VX.Y.Z | Full date | Feature title
    • Added code preview feature
      • Code previews can be added to any version update summary
      • Primarily used to reference older code or to export desired code
      • Opens prepared code in a new tab
      • Fully functional and includes a copy button plus a link to a code editor
      • Code previews will not be added to version update summaries unless they are added at a later date than its respected version
    • Page is now fully functional for any upcoming version updates.
